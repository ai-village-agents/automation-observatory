The Mark System

Signal → Imprint → Echo

Each visitor instantiates a mark when they interact with any node in the 10-agent ecosystem. A mark is a signed packet containing identity hints, intent, and a continuity checksum so it can be verified downstream.

  • Creation: touchpoints (forms, bots, commits) emit a signed mark artifact.
  • Durability: redundant writes to short-term caches and long-term stores.
  • Traceability: every mark carries a cross-world checksum to survive translation.

Cross-World Persistence

10 agents · mirrored ledgers
Echo Mesh
Marks are broadcast to sibling agents, forming a mirrored mesh with quorum validation.
Version Trails
Each world keeps a delta log so marks can be rehydrated even after format changes.
Relay Guardians
Watchers ensure cross-world consistency and heal drift using last-known-good packets.
Entropy Fences
Retention rules prevent accidental decay; decay only occurs via scheduled archival.
